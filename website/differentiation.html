i'll briefly go over what a derivative is, some examples of how to find it using a computer, and finally what i think is the fastest and most accurate way to calculate a derivative on a computer.

what is differentiation?

differentiation is the gradient of a graph. you have likely come across a gradient in school with a formula as the difference in two y co-ordinates per the difference in two x co-ordinates, where each x and y is taken from two points on a straight line as shown below. $$x_2-x_1 \over y_2 - y_1$$ this works perfectly but only for a straight line, what if we wanted to take this for an exact point on a continuously changing function? we can still use this equation but it will be very hard to get the exact value of the derivative of an exact point.
the solution that was found for this was to re-write the above simple definition of a derivative and make it more general with functions and limits. so \(y_1, y_2\) are now f(x) and the x's are still x's. for the difference we use limits, so we can theoretically find the exact point as one point is tending to the other. this equation is newtons quotient:

newtons quotient

$$ f'(x) = \lim_{h \to 0} \frac{f(x + h) - f(x)}{h} $$
one question you might have is why are there no x co-ordinates in the bottom of the fraction? well this is after simplifying it like this: $$ \frac{f(x + h) - f(x)}{(x + h) - x} $$ as you can see the x's cancel out.
from this we can now move onto our first basic form of differentiation.

finite difference method

there are a number of ways to figure out the derivative just from newtons quotient. the finite difference method is probably the most straightforward and well known method. where it is essentially just performing newtons-quotient on a function at a certain point. here is some example code for illustration:

class Differentiator:
    def __init__(self, func):
        self.func = func

    def finite_difference(self, x, h=0.001):
        numerator = self.func(x + h) - self.func(x)
        return numerator / h


def square(x):
    return x**2

def cube(x):
    return x**3

sq_diff = Differentiator(square)
print(f"Derivative of x^2 at 2: {sq_diff.finite_difference(2)}")

cube_diff = Differentiator(cube)
print(f"Derivative of x^3 at 2: {cube_diff.finite_difference(2)}")

So now we have to ask, how can we increase the accuracy of our derivative? One way to do this would be to hard code everything, but when thinking of every different type of way to differentiate it quickly becomes obvious that this would take a crazy amount of time. Along with this, when differentiating, complex problems can involve completing many rules many times, which can slow things down. But don't worry there is a solution to all of our differentiating with a computer problems, automatic differentiation.

Automatic Differentiation

Automatic differentiation (AD) is a way that we can differentiate a given function in a single step, and it will give an exact value for the derivative that you want. Making it as fast as possible whilst also being super accurate.

The first time automatic differentiation was written about and used was in the 1960s but nothing much came of it at the time, and only recently has automatic differentiation taken off and gained mass popularity. The reason for this sudden popularity is AI, where finding the derivative for back propagation became very important.

There are a few methods for AD, all with their pros and cons. The first that we can look at is forward mode AD.

Dual Number

Dual Numbers are a core concept for forward mode AD, so knowing them is essential. It is pretty simple and it is defined by these two rules: \begin{align*} \epsilon^2 &= 0 \tag{1} \label{epsilon rule 1}\\ \epsilon &\neq 0 \tag{2}\label{epsilon rule 2}\\ \end{align*} looking through these rules you quite quickly realise this is a complicated variable, as the only real number that equals 0 when squared is 0 itself, however \eqref{epsilon rule 2} says that it can't be this.
you can think of it as similar to the imaginary number i, where \(i^2 = -1\) makes no sense, as the \(\sqrt{-1}\) has no solution, due to no number being multiplied by itself being equal to -1
The most important thing about dual numbers is the following proposition $$ f(x + \epsilon) = f(x) + \epsilon f'(x) $$ where we can see that just by adding \(\epsilon\) to our variables we can get the derivative of any function!

here is a quick example of the theorem \begin{align} f(x) &= x^2 + 6x - 1 \\ f'(x) &= 2x + 6 \tag{*} \label{deriv} \end{align} \begin{align*} f(x + \epsilon) &= (x + \epsilon)^2 + 6(x + \epsilon) - 1 \\ &= x^2 + 2x\epsilon + \cancel{\epsilon^2} + 6x + 6\epsilon - 1 \\ &= (x^2 + 6x - 1) + 2x\epsilon + 6\epsilon \quad \text{group like terms}\\ &= f(x) + \epsilon \underbrace{(2x + 6)}_{\text{matches } \eqref{deriv}} \\ f(x + \epsilon) &= f(x) + \epsilon f'(x) \end{align*}

class ForwardAutoDiff:
    def __init__(self, real, dual):
        self.real = real
        self.dual = dual

    def __add__(self, other):
        real = self.real + other.real
        dual = self.dual + other.dual
        return ForwardAutoDiff(real, dual)

    def __mul__(self, other):
        real = self.real * other.real
        dual = self.real * other.dual + self.dual * other.real
        return ForwardAutoDiff(real, dual)

 
def diff(f, x):
    return f(ForwardAutoDiff(x, 1)).dual


def f(x):
    return x * x + ForwardAutoDiff(6, 0) * x + ForwardAutoDiff(-1, 0)


if __name__ == "__main__":
    x = 3
    print(diff(f, x))  # expected: 2x + 6 = 12
